FASE 1

Requerimientos funcionales

  • Gestión informativa pública.
  • Gestión de talento humano.
  • Gestión de procesos legales.

Requerimientos no funcionales

  • Usabilidad, interfaz intuitiva, navegación clara, accesible para usuarios sin conocimientos técnicos.
  • Seguridad.
  • Escalabilidad con capacidad de soportar aumento de usuarios y volumen de información.
  • Rendimiento para tener tiempos de carga óptimos.

Actores involucrados

  • Administrador del sistema: gestiona la plataforma, crea usuarios, válida publicaciones, controla accesos.
  • Profesionales de la firma: consultan y actualizan procesos, administran su perfil.
  • Profesionales externos (postulantes): registran su hoja de vida y hacen seguimiento al estado de la postulación.
  • Usuarios del público en general: consultan información de la firma, servicios, noticias y contacto.

Información a registrar

Información de la firma

  • Nombre, historia, misión, visión, trayectoria.
  • Servicios jurídicos.
  • Noticias y publicaciones legales.
  • Reconocimientos, alianzas, certificaciones.

Información de los profesionales

  • Datos personales (nombre, contacto, ubicación).
  • Perfil profesional (formación académica, experiencia, especialidad).
  • Hoja de vida (currículum cargado en PDF u otro formato).
  • Estado de vinculación (adscrito, en evaluación, rechazado).

Información de procesos legales

  • Identificador del caso.
  • Cliente asociado.
  • Profesional responsable.
  • Tipo de proceso.
  • Estado del proceso (activo, en trámite, cerrado).
  • Fechas importantes (inicio, audiencias, cierre).

Información de usuarios externos

  • Consultas realizadas por el formulario de contacto (nombre, correo, mensaje).

Paso

En el ámbito del desarrollo web moderno, la usabilidad y el diseño responsive se han convertido en pilares fundamentales para garantizar experiencias digitales accesibles, intuitivas y adaptadas a las necesidades de los usuarios. En este contexto, React se posiciona como una de las bibliotecas más utilizadas y robustas para la creación de interfaces de usuario dinámicas y eficientes.

¿Qué es React?

React es una biblioteca de JavaScript desarrollada por Facebook cuyo objetivo principal es la construcción de interfaces de usuario basadas en componentes reutilizables. Estos componentes permiten estructurar la aplicación en piezas pequeñas, lo que mejora la organización, la mantenibilidad y la escalabilidad del código.

Usabilidad con React

React facilita la implementación de prácticas de usabilidad gracias a:

  • Componentes reutilizables que garantizan consistencia en el diseño y en la interacción.
  • Virtual DOM, que optimiza el rendimiento y hace más fluidas las actualizaciones en la interfaz.
  • Manejo eficiente del estado con herramientas como React Hooks o bibliotecas externas (Redux, Zustand).
  • Integración sencilla con librerías de accesibilidad (ej. React Aria), lo que permite cumplir estándares como WCAG.

Diseño responsive con React

Si bien React no define directamente los estilos, ofrece flexibilidad para integrar herramientas que potencian el diseño adaptable:

  • CSS moderno: uso de flexbox, grid y media queries.
  • Frameworks de UI compatibles: Tailwind CSS, Material UI, Bootstrap, Chakra UI.
  • Server-Side Rendering (SSR) y Static Site Generation (SSG): con Next.js.
  • React Native: extensión del ecosistema para trasladar la lógica de componentes a aplicaciones móviles.
